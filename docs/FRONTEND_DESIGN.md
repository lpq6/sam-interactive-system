# Frontend Design Document
# SAM Interactive System - 前端设计文档

## 📐 设计理念

### 核心原则
1. **简洁至上** - 深色主题，减少视觉干扰，专注于图像标注
2. **即时反馈** - 每个操作都有明确的视觉响应
3. **专业工具感** - 类 Photoshop/Figma 的工具栏布局
4. **低学习成本** - 直觉化的图标和交互

## 🧩 组件设计

### Header 组件
- 固定顶部，z-index: 100
- 左侧 Logo + 标题 (渐变色文字)
- 右侧: 设备状态徽章 + 重置按钮

### Toolbar 组件
- 固定宽度 280px
- 纵向排列的工具区块
- 每个区块有小标题 (大写、小字、灰色)

#### 工具选择区块
```
┌─────────────────────────┐
│ 🔧 分割工具             │
│ ┌──────────┬──────────┐ │
│ │ 📍       │ ⬜       │ │
│ │ 点击分割 │ 框选分割 │ │
│ └──────────┴──────────┘ │
└─────────────────────────┘
```
- 2列网格布局
- 激活状态: 蓝紫色边框 + 半透明背景

#### 使用说明区块
- 根据当前工具动态显示操作提示
- 14px 字号，行高 1.6

#### 标记列表区块
- 仅在有点标记时显示
- 前景点: 绿色圆点 ●
- 背景点: 红色圆点 ○
- 显示坐标和类型

### Canvas Area 组件

#### 上传区域 (无图片时)
```
┌─────────────────────────────────────┐
│                                     │
│              🖼️                     │
│       上传图片开始分割               │
│  拖拽图片到此处，或点击选择文件      │
│     支持 JPG, PNG, WebP 格式       │
│                                     │
└─────────────────────────────────────┘
```
- 虚线边框 (2px dashed)
- hover 时边框变实线 + 浅色背景
- 拖拽时有放大动画 (scale: 1.02)

#### 画布 (有图片时)
- 图片等比缩放适应容器
- 最大宽度: 900px, 最大高度: 700px
- 双层 Canvas:
  - 底层: 原图
  - 顶层 (overlay): 分割掩码 (mix-blend-mode: screen)

#### 点标记
- 12px 圆点
- 前景: 绿色 + 白色边框 + 绿色发光
- 背景: 红色 + 白色边框 + 红色发光
- position: absolute, transform: translate(-50%, -50%)

#### 框选区域
- 2px 实线边框, 颜色: primary
- 半透明蓝色背景 (rgba(99, 102, 241, 0.1))
- 拖拽时实时更新尺寸

#### 加载遮罩
- 半透明深色背景
- 居中的 spinner 动画
- 文字: "SAM 正在分析..."

## 🔄 交互流程

### 图片上传流程
```
用户操作           系统响应              视觉反馈
─────────────────────────────────────────────────
拖拽图片    →    验证类型        →    边框高亮
          →    上传服务器      →    loading
          →    获取尺寸        →    渲染画布
          →    清除旧状态      →    空画布就绪
```

### 点击分割流程
```
用户操作           系统响应              视觉反馈
─────────────────────────────────────────────────
点击画布    →    记录坐标        →    显示标记点
(可多次)
点击"分割"  →    发送API请求    →    loading遮罩
          →    接收掩码        →    半透明叠加
          →    更新结果面板    →    显示指标
```

### 框选分割流程
```
用户操作           系统响应              视觉反馈
─────────────────────────────────────────────────
按下鼠标    →    记录起点        →    开始绘制框
拖拽       →    实时更新框      →    蓝色半透明框
释放鼠标    →    计算框坐标     →    框定格
          →    自动发送请求    →    loading
          →    接收掩码        →    叠加显示
```

## 📱 响应式设计

### 断点
- `> 1024px`: 完整三栏布局
- `768px - 1024px`: 缩窄侧边栏
- `< 768px`: 单栏布局，侧边栏堆叠到顶部

### 移动端适配 (未来)
- Canvas 支持触摸缩放
- 工具栏折叠为底部 Tab
- 结果面板改为弹出式

## ⚡ 性能优化

### 图片处理
- Canvas 渲染而非 DOM img
- 避免频繁的 toDataURL 调用
- overlay 使用独立 canvas 层

### 网络请求
- 上传使用 FormData + 进度
- 分割请求防抖 (连续点击时)
- 结果图片使用 base64 内联 (减少请求)

### 渲染优化
- React.memo 避免不必要的重渲染
- useCallback 缓存事件处理函数
- Canvas 绘制使用 requestAnimationFrame
